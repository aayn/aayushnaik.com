Honest revised lean: B · Astro

I led with A because it's lower-friction. But re-examining against your specific values — minimalist, dislikes redundancy, prefers static-on-CDN, taste for typographic precision — Astro fits better:

  • Zero JS on essay reads — readers' phones don't run a React runtime to display prose. Aligns with minimalism in a deep way.
  • Typed content schemas — your eight-level Kesselman ladder gets compile-time enforcement. confidence: hightly likely (typo) fails the build, not at render-time.
  • Migration cost is bounded — your homepage has 5 small components. ~half-day of tightly-scoped work. The interactive bits (clock, theme toggle) keep working as React islands.
  • Right shape for the next 5 years — you're starting an essay practice. The framework you commit to now matters more than today's friction.

Pick A if you want zero risk to the homepage and want to ship essays this weekend. Pick B if you'd rather invest the half-day to land on the right substrate for the long arc.
