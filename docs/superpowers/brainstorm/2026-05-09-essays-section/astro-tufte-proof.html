Astro · Tufte/Gwern compatibility, demonstrated

Astro outputs static HTML. You write whatever HTML, CSS, and components you want on top. There is no design opinion; there is no UX opinion; there is only "how files are organized and built."

Frontmatter validation (free)

// content.config.ts — runs at build time
import { defineCollection, z } from 'astro:content';

const essays = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    subtitle: z.string().optional(),
    importance: z.number().min(0).max(10),
    confidence: z.enum([
      'certain', 'highly likely', 'likely',
      'possible', 'unlikely', 'highly unlikely',
      'remote', 'impossible',
      'log', 'emotional'
    ]),
    status: z.enum([
      'notes', 'draft',
      'in progress', 'finished'
    ]),
    stance: z.enum([
      'agree', 'disagree', 'withholding'
    ]).optional(),
  }),
});

Every feature we've designed, mapped to Astro

metacommentary block Read frontmatter in the layout, render the monospace meta grid. Plain HTML/CSS.
importance/confidence/status/stance Typed by zod schema; build fails on typos (stance: agreee caught at build, not at runtime).
right-rail sidenotes (desktop) CSS Grid in EssayLayout.astro. Pure CSS — no framework involved.
mobile inline-expand notes ~1 KB vanilla JS in a <script> tag inside Sidenote.astro. No React needed.
position: sticky close ✕ Pure CSS. Astro doesn't touch CSS — it just emits whatever you write.
scroll restoration on close Same vanilla JS handler. ~10 lines.
dropcaps .essay-body p:first-of-type::first-letter in CSS. Zero JS.
glossary page src/pages/essays/glossary.astro. Just another page.
meta-block labels link to glossary The MetaBlock.astro renders <a href="/essays/glossary#confidence"> on each label. Pure HTML.
Kesselman ladder graphic on glossary HTML + CSS. Same component you saw on screen earlier; ports as-is.
theme toggle (dark/light) Existing React component as <ThemeToggle client:load />. Hydrates only this island.
ambient clock canvas Existing React component as <AmbientClock client:idle />. Same code; islanded.
typography (Iowan, Source Serif, Plex Mono, Space Grotesk) Self-hosted woff2 + @font-face in global CSS. Astro is invisible here.
tailwind v4 @astrojs/tailwind integration. Plug-and-play.
future weird thing you want Drop in any HTML/CSS/JS, any React/Svelte/Vue island, any custom MDX component. Astro doesn't constrain.

Astro is a build tool, not a UX framework. It cares about how things get to the browser, not what things look like once they're there. Tufte/Gwern aesthetics live in the HTML/CSS/components — none of which Astro touches.
