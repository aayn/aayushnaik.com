The mechanic, written out

  • one button   The ✕ is a child of the note element with position: sticky; top: 12px; right: 12px. It never "moves out" of the note — sticky just lets it pin to viewport top while its parent scrolls past.
  • three sticky behaviors, automatic   (1) note in view → ✕ at note's corner. (2) note's top scrolled past → ✕ stuck to viewport top. (3) note's bottom reaches viewport top → ✕ unsticks and scrolls away with note (you've finished reading anyway).
  • insertion point   end of the sentence containing the marker; never paragraph break, never mid-sentence.
  • open scroll   smooth-scroll so note's top sits ~30% down from viewport top.
  • close paths   tap ✕, or re-tap the original marker (highlighted while open).
  • close scroll   scroll position restores so marker is at the same viewport y-position it was when tapped.
  • multiple notes   allowed; each independent.
