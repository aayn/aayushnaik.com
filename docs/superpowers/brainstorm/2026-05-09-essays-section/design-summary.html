Design summary · everything settled

Purpose

sectionA new /essays section, anti-performance counterpart to Anticynical. Slower writing, with epistemic disclosure attached.
spiritA workshop, not a stage. Notes, drafts, in-progress positions all welcome. Form is part of the medium (Tufte/Gwern lineage).
anticynicalPaused. Future relationship TBD; possibly newsletter or lead-feed for finished pieces.

Information architecture

URLs/essays, /essays/<slug>, /essays/glossary
homepageNo link from homepage until content exists. Seed first; link later.
nav"← essays" on essay pages; "← home" on essays index. No top-bar nav.

Visual identity

paletteDark-first (warm #0c0a08). Light/dark toggle preserved.
bodySerif — Iowan Old Style / Source Serif 4 / Palatino fallback. ~14.5px desktop, 1.62 line-height.
chromeSpace Grotesk light — eyebrow, sidenote text, "← essays" links, theme toggle. Matches homepage.
meta blockIBM Plex Mono — gives the metadata a "specification sheet" feel.
titlesSerif, light weight, slightly tightened tracking.
italicSerif italic for subtitles + emphasis.
densityCompressed Tufte — tighter leading and narrower body than canonical Tufte.

Metacommentary fields

importance0–10 integer (Gwern decile)
confidenceKesselman 8 + special: certain · highly likely · likely · possible · unlikely · highly unlikely · remote · impossible · log · emotional
statusnotes · draft · in progress · finished
stanceagree · disagree · withholding (or — for log entries)
datesAuto from git: created · modified. Shown small under title, not in meta block.
validationZod schema in Astro content collections — typos fail at build, not runtime.

Essay page layout · desktop

gridBody ~620px left · 220px right rail · ~36px gap
orderTitle → subtitle → date line → meta block → body w/ dropcap
sidenotesRight-rail, always visible. Numbered ¹²³; markers in body, full text in rail anchored near anchor line.
dropcapFirst letter of first paragraph; CSS ::first-letter

Essay page · mobile mechanics

sidenoteTap marker → note inserts at end of containing sentence; smooth-scroll to ~30% from viewport top.
closeOne button. position: sticky on ✕ — sits in note's top-right; sticks to viewport top while note is being read.
close pathsTap ✕ or re-tap marker. On close, scroll restores so marker is at original viewport y-position.
typeBody 14.5 → 16.5px; line-height 1.62 → 1.70.
chrome"← essays" → "←" only; eyebrow shrinks; theme toggle preserved.

Index page

layoutInline meta strip — title → italic subtitle → mono meta row.
colorStatus as subtle chip (green=finished, cream=active, dim=draft/notes); stance tinted (green-agree, red-disagree, neutral-withholding).
sortChronological, newest first. No filters / no categories initially.
page header"Essays" — italic sub: "Beliefs and positions, weighed and dated" (placeholder; rewriteable)

Glossary

URL/essays/glossary
sections#importance · #confidence · #status · #stance
graphicKesselman ladder embedded in confidence section (the chart you saw).
linkingEach label in essay meta blocks links to its glossary anchor — readers can click "confidence" to learn what "highly likely" means.

Architecture

frameworkAstro — content-first, static-by-default, zero JS for prose
authoringMDX files in src/content/essays/<slug>.mdx
routingFile-based — src/pages/essays/[slug].astro
reactExisting AmbientClock + ThemeToggle as client:* islands. Zero refactor of their internals.
CSSTailwind v4 retained; component-scoped CSS where appropriate.
deployStatic HTML to CDN (your hosting preference).
